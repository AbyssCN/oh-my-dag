## 0. 一句话

把 investigate 的根因调查纪律用 DAG 引擎扇成**并发多假设验证**:一假设一 leaf、skeptic 跨模型证伪、judge 收敛,codegraph(有则用)锁范围。**独立新增,不替代 sast/verify/investigate**。

## 2. 架构:plan-based(D-2,owner 已拍)

核心并发上真 DAG 引擎(`runExecutorDagWithPlan`),非手编排脚本。两处对 issue #9 字面的**刻意偏离**(读代码逼出的反-happy-path 发现):

- **D-3 三振循环手写,不用 `iterateExecutorDag`**。后者是 LLM-conductor 每轮重规划路径,不吃预构造 plan;`runDebug()` 手写 `for round in 1..3` 包 `runExecutorDagWithPlan(compileDebugPlan(...))`,与 `arch/deepen-plan` 预构造姿态一致。
- **D-4 repro/oracle 走裸 `$` 不走 command 节点**。引擎 `executor:'command'` 是 fail-closed 白名单,用户 `--repro` 传的命令过不了闸。复现(拿 red)+ oracle 复验(red→green)在 `runDebug()` 裸 shell 跑;**map 假设扇出 + verify + judge 核心才上引擎**——DAG 用在真正需要并发处。

### 2.1 引擎形状

```
runDebug()                                   [src/harness/debug/run-debug.ts]
 ├─ red = $`reproCmd`         拿确定失败证据(裸 shell,--repro 给才跑)
 ├─ cgAvailable = probe codegraph   plan 期能力探测 → 喂 scope goal(D-5 降级)
 └─ for round in 1..3:
      plan = compileDebugPlan({failure, red, cgAvailable, priorRefuted})  [debug-plan.ts]
      res  = runExecutorDagWithPlan(plan, config)
      judged = res.results['judge']
      judged.confirmed? → break → 报告 + 提议最小修(默认不改文件)
      else priorRefuted += 本轮 REFUTED 假设 → 下轮带反馈重扇
    3 振全 REFUTED → 停,升 owner(报告排除了什么,不无限烧 token)
```

### 2.2 compileDebugPlan 节点(预构造 ConductorPlan,照 buildDeepenPlan)

| node id | executor | 内容 | 新/复用 |
|---|---|---|---|
| `scope_lock` | `agent` | 只读锁范围:goal 按 cgAvailable 指「符号→codegraph impact/callers」或「文本→ugrep/read」。复用 agent-leaf 双路 | ♻️ goal 措辞 |
| `hypotheses` | `map` | lister=**假设生成器**(读 failure+red+scope,emit `{hypotheses:[{id,claim,where}]}`)→ 每假设一 verify-leaf(skeptic:观察→推论→验证,引 file:line,默认 REJECTED 除非证据支撑)。depends_on: [scope_lock]。maxItems=5 | 🆕 lister goal + ♻️ skeptic 框 |
| `judge` | `leaf` | depends_on: [hypotheses]。择 CONFIRMED 根因(证据最硬)+ 提议最小修文本;无 CONFIRMED → 输出「无根因确认」驱动三振 | ♻️ |

收口 `PlanSchema.parse(...)`(图坏在构造期炸,不流进执行器)。scope_lock goal 刻意避 producesFiles 强写信号(实现/创建/写入/生成 + 文件后缀),防只读叶被产物闸误判 empty-done failed(deepen 同款守卫)。

## 3. 反-happy-path 约束(验收判据)

1. **codegraph 非前置**(D-5):probe 探到→符号导航,探不到→ugrep 降级,精度降照跑。绝不硬依赖。
2. **无根因不修**:judge 未 CONFIRMED → 不提修,回 fanout。
3. **默认只提议不改文件**:judge 输出提修文本,无 agent 写盘。实装修复走显式 gated 模式(v1 不做,`--fix` 留 future)。
4. **修复必过 oracle 复验**:仅 gated fix 模式相关(v1 defer);oracle command `red→green` 由裸 shell 确认,不靠 LLM 自称修好。
5. **3 振全证伪 → 停升 owner**,不无限扇出。
